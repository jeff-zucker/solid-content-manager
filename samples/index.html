Welcome to Solside

a simple IDE and file manager for Solid

Solside provides a simple way to browse, edit, and manage files and folders on a Solid data store. You are using it now. You can see a directory listing of the Solid data store this app lives in on the left. Feel free to explore the files and directories. If you were me, you could edit the files. If you don't already have one, get your own data store, and use Solside to login and edit your files. When you've had a chance to use it, please send feedback.

  • open a folder : click on its name
  • view, edit, or cut and paste from a file : click on its name
  • view any Solid URL : enter the URL in the input bar
  • download a file : right click on the file name
  • login, logout, go home, edit profile, set editor prefs : options button
  • toggle viewing data only, editor only, split screen : view button
  • save editing : click on the save edits button
  • cancel editing : open any other file or folder
  • create new file or folder, upload file(s) : click on the current folder's icon
  • delete file : click on the file's icon
  • delete folder : move into the folder, empty it, click on its icon

About the Data Browser

The data browser is an iframed view of the Solid URL so, is exactly what you'd see going to the same URL directly and has all the same functionality. This means you always see your edits the way everyone else will see them and also that you have access to all of the tools in the data browser.

Navigating Solid data

If you navigate using the file menu or URL bar, the data browser will be refreshed each time and therefore always be in sync with the file manager. If you navigate using the data browser, however, it will become out of sync with the file manager. Selecting a new resource in the file manager will bring the data browser back in sync. Or if you want to sync in the other direction, right click any url in the data browser, copy the url and paste it into the file manager's URL bar. That will make the file manager catch up to your data browser location.

About Access Rights

Although I am working on an ACL-aware version, the current version does only a simple owner check. Rights are always visible in the interface: the file/folder icons are only green if you have rights to Control the folder; the save-edits button is only green if you are visiting a resource for which you have Write privileges; the options button is only green if you are logged in.

About logins

The logins for Solside and the iframed data browser are separate, log in or out of both or either as desired. The login is a popup so check your popup blocker if you don't see it.
